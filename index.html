Git and GitHub Overview

  • Git is a version control system, a method of interacting with documents (computer code, typically) that allows single or multiple authors to foreground changes made to those documents. Different versions of documents can then be easily compared and the authorship of version's changes can be easily ascribed.
  • GitHub, although wholly dependent upon and interrelated with Git, should be defined separately. It is a repository for files that are being manipulated using Git. It serves both as a public and private repository, allowing its users to collaborate and share documents and code with each other and with the world, if they choose.
  • It is possible to Git without GitHub, either locally or using a third-party host, but becasue GitHub is designed to work hand-in-glove with Git, most people choose to use GitHub.

Did you know?

You're not alone if you find Git intimidating. Git was originally created ten years ago by Linus Torvalds, the inventor of Linux, to track changes to the Linux kernel. Initially, it received a somewhat skeptical response, as even Google's engineers claimed it was "expressly designed to make [us] feel stupid."

Git: essential commands

The following is for my own benefit as I repeatedly forget the correct order of git commands when committing changes to GitHub.

Steps for committing changes to GitHub:

  • Make local changes to your document (for our purposes, we assume it's index.html)
  • In GitBash or your local bash equivalent, make sure you're working in the correct branch by typing the following: git checkout [yourbranchname]. Yes, I have multiple branches already because I am dumb.
  • Stage the file you want to commit by typing the following: git add index.html
  • Commit the document with a brief comment by typing the following: git commit -m "note your changes inside these quotation marks"
  • Push the committed, commented document to the correct branch on GitHub by typing the following: git push -u origin [yourremotebranchname]. If pushing to the "master," you would type: git push -u origin master.To follow up, run git status and then double-check your GitHub repositories.
  • For this assignment, I first created the repository SemanticTags through GitHub's web interface, then used the following commands in GitBash:
    git remote add origin git@github.com:morgan-lang/SemanticTags.git
    followed by
    git add index.html
    followed by
    git commit -m "added preview link"
    and finally by
    git push -u origin master
    The first command identified SemanticTags as a valid origin (remote) repository, the second command staged index.html, the third command "committed" the changed document with a comment regarding its altered content, and the final "push" command finally uploaded it to github, tracking its changes.

    NOTE: to publish your work to Github's gh-pages, use the following sequence:


    git add .
    git commit -m "merging to gh-pages"
    git checkout gh-pages
    git merge master
    git checkout master
    git push -u origin master
    git push -u origin master:gh-pages
